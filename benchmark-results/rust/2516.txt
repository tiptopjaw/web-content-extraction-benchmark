{"title":"GHC 9.14.1 is now available","author":null,"date":null,"main_content":"zubin - 2025-12-19\n\nThe GHC developers are very pleased to announce the release of GHC 9.14.1.\nBinary distributions, source distributions, and documentation are available at\ndownloads.haskell.org.\n\nGHC 9.14 brings a number of new features and improvements, including:\n\nSignificant improvements in specialisation:\n\nThe SPECIALISE pragma now allows use of type application syntax\n\nThe SPECIALISE pragma can be used to specialise for expression arguments\nas well as type arguments.\n\nSpecialisation is now considerably more reliable in the presence of\nnewtype s\n\nSignificant GHCi improvements including:\n\nCorrectness and performance improvements in the bytecode interpreter\n\nFeatures in the GHCi debugger\n\nSupport for multiple home units in GHCi\n\nImplementation of the Explicit Level Imports proposal\n\nRequiredTypeArguments can now be used in more contexts\n\nGreatly improved SSE/AVX2 support in the x86 native code generator backend\n\nInitial native code generator support for LoongArch\n\nThe WebAssembly backend now supports evaluation via the interpreter, allowing\nGHCi and TemplateHaskell evaluation, including foreign import javascript\nusage from within the browser\n\nA new primop annotateStack# for pushing arbitrary data onto the call stack\nfor later extraction when decoding stack traces\n\n-Wincomplete-record-selectors is now part of -Wall. Libraries compiled\nwith -Werror may need adjustment.\n\nA major update of the Windows toolchain\n\nImproved compatibility with macOS Tahoe\n\n… and many more\n\nA full accounting of changes can be found in the release notes.\nSee the migration guide for guidance on migrating programs to this release.\n\nNote that while this release makes many improvements in the specialisation\noptimisation, polymorphic specialisation remains disabled by default in the\nrelease due to concern over regressions of the sort identified in\n#26329. Users needing more aggressive specialisation can explicitly\nenable this feature with the -fpolymorphic-specialisation flag. Depending\nupon our experience with 9.14.1, we may enable this feature by default in a\nlater minor release.\n\nGHC development is sponsored by:\n\nJuspay\n\nQBayLogic\n\nChannable\n\nHaskell Foundation\n\nSerokell\n\nWell-Typed\n\nTweag\n\nDotcom-Monitor\n\nLoadView\n\nWeb Hosting Buddy\n\nFind My Electric\n\nStandard Chartered\n\nUpCloud\n\nWe would like to thank these sponsors and other anonymous contributors\nwhose on-going financial and in-kind support has facilitated GHC maintenance\nand release management over the years. Finally, this release would not have\nbeen possible without the hundreds of open-source contributors whose work\ncomprises this release.\n\nAs always, do give this release a try and open a ticket if you see\nanything amiss."}
