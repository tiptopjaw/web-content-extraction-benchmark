50..= 50 ), y + rng.gen_range ( - 50..= 50 )),\n( x + rng.gen_range ( - 50..= 50 ), y + rng.gen_range ( - 50..= 50 )),\n( x + rng.gen_range ( - 50..= 50 ), y + rng.gen_range ( - 50..= 50 )),\n],\n// Random RGBA color (RGB + Alpha for transparency)\ncolor: [\nrng.gen_range ( 0..= 255 ), // Red\nrng.gen_range ( 0..= 255 ), // Green\nrng.gen_range ( 0..= 255 ), // Blue\nrng.gen_range ( 0..= 255 ), // Alpha (transparency)\n],\nimg_width,\nimg_height,\n}\n}\n\n/// Apply a random mutation to this triangle\n///\n/// # Arguments\n/// * `sigma` - Mutation strength (0.0-2.0, default 1.0)\n///\n/// `&mut self` means:\n/// - `&` = borrowed (we don't take ownership)\n/// - `mut` = mutable borrow (we can modify the triangle)\n/// - `self` = this is a method that operates on an instance\npub fn mutate ( & mut self, sigma: f32 ) {\n// Import the weighted index distribution for selecting mutation types\nuse rand:: distributions:: WeightedIndex;\nuse rand:: prelude:: *;\n\nlet mut rng = rand:: thread_rng ();\n\n// Weighted selection of mutation type\n// Higher weights = more likely to be selected\n// [30, 35, 30, 5] means Shift and Color are less common, Point is most common, Reset is rare\nlet weights = [ 30, 35, 30, 5 ];\nlet dist = WeightedIndex:: new ( & weights ).unwrap ();\n\n// Sample from the distribution to pick a mutation type\n// `match` is Rust's pattern matching - Here you simply use.par_iter_mut() from the rayon crate (which packages/libraries are called in Rust) on a list and that’s it.\n\n/// Evaluate fitness for all individuals in parallel\n///\n/// This is where Rust really shines!\n/// We're going to evaluate ALL individuals simultaneously across all CPU cores.\n///\n/// **Rust Concept: Parallel Mutation**\n/// - `.par_iter_mut()` creates a parallel iterator over mutable references\n/// - Each thread gets exclusive access to different individuals\n///\n/// **Why is this safe?**\n/// 1. Each individual is independent (no shared state)\n/// 2. `&mut` ensures exclusive access per thread\n/// 3. When using agentic coding I’ve noticed that relatively straightforward functionality is often generated as code, rather than pulling in an existing package. The function.par_iter() from the rayon library makes this again multi-threaded to leverage all available cores in your machine during this step.\n\n/// Parallel version of image_diff (faster for large images)\n///\n/// **Rust Concept: Fearless Concurrency**\n/// This function uses ALL your CPU cores automatically, safely!\n///\n/// How does Rust make parallel programming safe?\n/// 1. The borrow checker prevents data races at compile time\n/// 2. Only immutable references (&) are used, so parallel access is safe\n/// 3. No locks, no mutexes needed - the type system guarantees safety\n///\n/// **When to use this?**\n/// - Large images (>1000x1000 pixels) - overhead is worth it\n/// - When you have multiple CPU cores (which you probably do!)\n/// - The population evaluation (comparing many images) benefits hugely\npub fn image_diff_parallel ( source: & RgbaImage, target: & RgbaImage ) -> f64 {\nassert_eq! (\nsource.dimensions (),\ntarget.dimensions (),\n\"Images must have same dimensions\"\n);\n\n// The ONLY difference: collect into vectors and use par_iter!\n//\n// **What's.par_iter()?**\n// - Creates a parallel iterator over a collection\n// - Rayon automatically splits work across CPU cores\n// - Uses work-stealing: idle cores help busy cores\n// - No manual thread management needed!\n//\n// We need to collect both iterators first since rayon's zip needs both sides parallel\nlet source_pixels: Vec < _ > = source.pixels ().collect ();\nlet target_pixels: Vec < _ > = target.pixels ().collect ();\n\nlet total_diff: u64 = source_pixels\n.par_iter () // <-- Parallel iterator over source pixels\n.zip ( target_pixels.par_iter ()) // <-- Zip with parallel iterator over target pixels\n.map (|( s, t )| {\nlet dr = ( s [ 0 ] as i32 - t [ 0 ] as i32 ).abs () as u64;\nlet dg = ( s [ 1 ] as i32 - t [ 1 ] as i32 ).abs () as u64;\nlet db = ( s [ 2 ] as i32 - t [ 2 ] as i32 ).abs () as u64;\ndr + dg + db\n})\n.sum (); // Rayon's sum() automatically combines results from all threads\n\ntotal_diff as f64 / ( source.width () * source.height () * 3 ) as f64\n}\n\nStep 4: The CLI\n\nOne thing I found Claude Code to be very useful for, is to create a CLI for applications.
