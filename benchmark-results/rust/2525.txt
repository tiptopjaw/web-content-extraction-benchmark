{"title":"Homelab projects to try this weekend (December 19 - 21)","author":"Patrick Campanale","date":"2025-12-19T18:30:15+00:00","main_content":"Are you ready to spin up some core services that every homelab should have? The three projects I’m showcasing this weekend all focus on core services in my own homelab, so here’s why you should run them too.\n\nHTG Wrapped 2025: 24 days of tech\n\n24 days of our favorite hardware, gadgets, and tech\n\nBy \nWill Verduzco\n\nTake your smart home to the next level with Home Assistant\n\nLocal control, anyone?\n\nIf you’ve not heard of Home Assistant yet (or maybe just haven’t taken the time to set it up), this weekend is the perfect time to dive in.\n\nHome Assistant allows you to do quite a few things within your smart home. For starters, it’s a fully self-hosted local control platform. With Home Assistant, you’re able to send commands to devices locally on your network instead of sending the commands through remote servers. This means when Cloudflare or AWS goes down, your smart home stays up and functional.\n\nAnother huge benefit to Home Assistant is how it brings so many different ecosystems together. I use Home Assistant to bring non-HomeKit devices into the Apple Home app, giving me Siri control and the ability to use Shortcuts to automate devices around my house.\n\nHome Assistant is one of those services that takes just a few minutes to get up and going in your homelab, but takes forever to master. I’ve been running Home Assistant for several years, but still only use a fraction of the features that it offers.\n\nSo, if you haven’t spun up a Home Assistant VM (or Docker container ) yet, then make that your task for the weekend.\n\nOrganize that stack of papers in your drawer with paperless-ngx\n\nWho needs papers when digital organization is so easy?\n\nYears ago, I used to pay for Neat to organize my scanned documents and make them searchable. These days, I just self-host paperless-ngx. This simple Docker container is all you need to lead a minimal, organized digital life.\n\nEssentially, once paperless-ngx is up and running, you just scan documents to your computer and then upload them to the paperless server—it handles everything else from there. With paperless, you’ll be able to organize and index your scanned documents. OCR ( optical character recognition ) tech is employed here to make uploaded documents searchable, too. Paperless even uses machine learning to automatically add tags and more to your uploaded documents, making it easier to skim or filter down to exactly what you’re looking for.\n\nOne of my favorite features is email processing, meaning you can send a document to an email address that the paperless server monitors, and it processes the image from there. Once processed, it can be deleted, marked as read, or do whatever else you want with the email. This makes sending a document when you’re on the road simple, without having to set up reverse proxies or own a domain.\n\nGet yourself ready for 2026 by spinning up paperless-ngx this weekend so you can finally start going through that drawer of papers you keep saying you’ll clean out.\n\nDocument your homelab setup with BookStack\n\nYou don’t think you need homelab documentation until you do.\n\nI’ve seen a lot of people talk about documenting their homelab, and I always thought, “Why?” It just seemed like something that I had no need for. I was wrong.\n\nRecently, I was working on a pretty in-depth project in my homelab that required Tailscale integration alongside quite a few custom configuration changes on multiple computers to work properly. Needless to say, I spent about six hours working to get everything functional, and I knew that I wouldn’t want to have to do all the research a second time.\n\nThat’s when I finally broke down and deployed a BookStack instance on my server. While you can use software like Obsidian to make a wiki, I wanted something that was a little more tailor-made and also web-based.\n\nWithin a few minutes, I was able to get the core documentation written for the service that I had just spent hours working on, and now if I need to deploy it again in the future, I have my own tutorial to follow.\n\nBookStack is basically a wiki for your homelab. It’s designed to be 100% self-hosted, and you can even open it up with a reverse proxy to the web if you want others to be able to access your wiki too. I have mine reverse proxied, just in case I want to write any public documentation that I don’t want on GitHub for any reason.\n\nDon’t be like me and wait until you need documentation to try and figure out how you’re going to write it. Just deploy the BookStack server this weekend and start documenting the little things, then, when big things come along, it’ll be there and ready for you.\n\nIf you’re just starting out with homelabbing, don’t worry, it won’t be long until it becomes the most useful part of your smart home. I use my homelab every single day for various purposes, and the list grows by the hour sometimes.\n\nJust a few months ago, I was sitting at about 20 Docker containers running on my main server. Now, I’m up to 50, and I don’t see it slowing down anytime soon. So, here’s to your homelab and all the projects you’ll create in it over the years."}
