{"title":"Polyglot NixOS: The Same Disk Image for All Architectures","author":"julian","date":"2025-12-19T00:00:00+00:00","main_content":"Recently a colleague mentioned building\nNixOS images that run unchanged on multiple architectures. Given the past\nadventures on this blog with systemd-repart and\ncross-compiling NixOS, I decide to give this a\ngo.\n\ntl;dr You can find a quick’n’dirty implementation\nhere. Check the repo for\ndetails on how to build and run it.\n\nSo do we want to do: We want to build one disk image that boots on\nx86_64, ARM AArch64, and RISC-V 64-bit. We limit ourselves here to\nUEFI platforms, which makes this pretty straight forward.\n\nFrom a high-level we need to:\n\nHave a NixOS configuration.\n\nBuild the system closure for each target.\n\nThrow everything into one /nix/store partition.\n\nPopulate the ESP to\nboot the right closure depending on the architecture.\n\nAll of this is surprisingly straight-forward. The ESP has\narchitecture-dependent default filenames for what the firmware should\nboot, given no other configuration. This means we can build an\nUKI per\narchitecture and drop it at the right place in the ESP\n( /EFI/BOOT/BOOTX64.EFI for 64-bit x86) and we are done!\n\nBy linking the system’s UKI in these locations on the ESP, we skip\nover having an actual bootloader and thus can’t have multiple\ngenerations, but it makes for a much leaner example!\n\nThe example repo puts the\nclosure for each architecture in a single Nix store partition. I\nthought this would bring some space savings, because files that are\nnot binary code should be largely the same. This doesn’t really pan\nout in this small example and we only save a couple of percent. Maybe\nit makes a bigger difference for larger closures.\n\nIf you want to dig into the details, the example\nrepo has the instructions\nhow to build and boot the image. I’m also eager to see someone\nbuilding a more comprehensive version of this that includes a fully\nfunctioning bootloader and multiple generations!\n\nQuick and Dirty Website Change Monitoring 👈"}
