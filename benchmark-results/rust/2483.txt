Can I use Node or Deno?\n\nJosh Derocher-Vlk\n\nCommunity Member\n\nYou've taken a look and ReScript and you want to try it out, but how do you get started? There's the installation page in the docs,\nwhich is great if you want to set up a new React app using create-rescript-app. There's instructions on how to add it to an existing project or set it up manually.\nBut that doesn't really answer the question \"Can I use this with X?\".\n\nYou can use ReScript anywhere you can use JavaScript\n\nReScript is just a language that compiles to JavaScript. Unlike other language like Elm or PureScript ReScript doesn't have a recommended framework or independent ecosystem, it's just part of the normal JavaScript world.\n\nHere's a really basic example that you can run in Node after compiling:\n\nRES\n\nConsole.log( \"Hello\" )\n\nJust run node index.res.js and you'll see \"Hello\" logged to the console. You can import compiled ReScript into any project that could import JavaScript.\nIf you can use.js or.mjs files, you can use ReScript. This does mean that languages with different file formats like Vue or Svelte require you to import the compiled JavaScript instead of writing it directly in the.vue or.svelte files.\n\nReal world projects are more than JavaScript files that you write; they use libraries and frameworks. This is where bindings come into play.\nA binding is a way to tell ReScript the types and imports from external JavaScript. You can think of bindings in the same way that you need to create a *.d.ts file to add types to a JavaScript library that doesn't use TypeScript.\n\nReScript has great integration with React and those bindings are kept up to date by the core team, but that doesn't mean you don't have other options!\n\nUsing existing bindings\n\nWhile ReScript isn't as large as TypeScript it has a small but growing list of bindings you can find on NPM. The website has a package explorer you can use to find official and community maintained bindings.\nMany major libraries have existing bindings. Here's a small set of what you can find.\n\nUsing libraries and frameworks created for ReScript\n\nBindings are great if you want to work with libraries written with JavaScript, but there are great options for libraries and frameworks written with ReScript, which means you don't need bindings.\n\nReScript Schema - The fastest parser in the entire JavaScript ecosystem with a focus on small bundle size and top-notch DX.\n\nrescript-relay - This is an amazing way to connect React to Relay and GraphQL\n\nrescript-rest - Fully typed RPC-like client, with no need for code generation!\n\nrescript-edgedb - Use EdgeDB fully type safe in ReScript. Embed EdgeQL right in your ReScript source code.\n\nResX - A ReScript framework for building server-driven web sites and applications.\n\nCreating your own bindings\n\nAt some point you will probably have to use a library that doesn't have bindings available. Asking on the forum is a great place to start. Someone else might have bindings already in a project that they just haven't published to NPM.\nYou can also get help and guidance on how to write bindings for what you need. Usually you can figure out what you need from looking at a libraries official docs.\nYou don't need to write bindings for an entire library, or even for all of a functions arguments. Just write what you need as you go.\n\nLet's take a look at the format function from date-fns. We can see the arguments in the docs, and how it should be imported and used.\n\nTS\n\nfunction format (\ndate: string | number | Date,\nformatStr: string,\noptions?: FormatOptions,\n): string;\n\nimport { format } from \"date-fns\";\n\nconst result = format ( new Date ( 2014, 1, 11 ), \"MM/dd/yyyy\" );\n\nThat's all we need to know to write bindings to use this function in ReScript.\nThe first thing we need to figure out is how to handle the type for what date-fns considers to be a date, which is Date | string | number. In ReScript things can't just be of different types like they can in JavaScript or TypeScript. There are a couple options here; you can make a function for each type such as formatString and formatDate, or you can create a variant type to map to the possible input types.\nCreating a function for each type is simpler, and it's most likely how you will use the library in your project. You probably have a standard type for Dates already. We'll also need a type for FormatDateOptions in case we want to pass options. We'll use labeled argmuments for our binding.\n\nRES\n\ntype formatDateOptions\n\n@module(\"date-fns\")\nexternal formatString: (\n~date: string,\n~formatStr: string,\n~options: formatDateOptions =?,\n) => string = \"format\"\n\nNow we can use the function!\n\nIf we need to use FormatDateOptions we can add to our type definition as needed. The first option is firstWeekContainsDate which can either be 1 or 4.\nHere's how we could write bindings for that.\n\nRES\n\n@unboxed\ntype firstWeekContainsDate =\n| @as(1) One\n| @as(4) Four\n\ntype formatDateOptions = {firstWeekContainsDate: firstWeekContainsDate}\n\nAnd when we use it it will output either 1 or 4.\n\nYou can write new bindings and extend existing types as you need.\n\nHow can I get started?\n\nYou can follow this guide to add ReScript to an existing JavaScript project to get a feel for how the language works and interacts with JavaScript.\nThe forum is also a great place to ask questions! Feel free to drop by and ask how to get started with a specific framework or project that you want to work on,\nand you'll probably get great advice and information from users who have already used ReScript for something similar.\n\nHappy coding!\n\nWant to read more?
