What many have come to learn the hard way is, this can lead to many unanticipated changes or assumptions, leading to surprise behaviours or hallucinations from the generative model. Providing a prompt such as write unit tests for the OrderServiceController within the context of a codebase would imply that it would write unit tests for said OrderServiceController. They pull in dependencies which don’t exist, or rewrite entire modules because it doesn’t comply with their ideal result to your prompt; they rewrite the same function five times before opting to rewrite the entire package.\n\nThat’s where understanding and employing a lifecycle which allows us to provide context, answer questions and even go over design and concepts before any code has been changed allows for a more effective partnership with AGI tools. Those agent-enabled editors mentioned support this entire lifecycle out of the box thanks to their chat-like widgets with the ‘assistant’ using one of three modes that can be used: Ask, Edit, and Agent.\n\nAsk\n\n“Ask mode for chat is optimized for asking questions about your codebase, coding and general technology concepts. Ask mode is particularly useful for getting a better understanding of your codebase, brainstorming ideas, and getting help with coding tasks.”\nVisual Studio Code\n\nOntop of those listed, I find ask utterly fantastic for learning a new codebase, troubleshooting an error within a module, and planning how to approach my task within the file. We can use Ask to essentially chat with our AI assistant without any codebase changes occurring, allowing developers to “go over” their thinking and design, revising and prompting for additional insights or known issues as if you were pair programming. By going over intention, design and behaviour prior to using Edit or Agent mode, we can benefit from outcomes like:\nA concrete understanding of the proposed changes\nDetails are more defined for the AGI to follow, leading to less hallucination potential or surprises.\nThe ability to catch conflicting concepts before anything has been changed\n\nEdit\n\n“With [chat] edit mode in Visual Studio Code, you can use natural language to make edits across multiple files in your project. Use this when you have small changes within a single file, such as updating the behaviour of a function or writing new test cases, or improving the perfect README.md. The outcomes from this mode should be pretty clear:\nVisible, easy to review changes to your file\nLeverage Generative AI to update and improve a document, section, etc.\n\nAgent\n\n“Agent mode is optimized for complex coding tasks based on high-level requirements that might require running terminal commands and tools. The AI operates autonomously, determining the relevant context and files to edit, planning the work needed, and iterating to resolve issues as they arise.”\nVisual Studio Code\n\nAgent truly shows the power, danger and potential fallacy of giving an AGI access to an entire local codebase. More impressively, it can also interface with Model Context Protocol (MCP) servers to carry out more complicated and intertwined tasks between services.
