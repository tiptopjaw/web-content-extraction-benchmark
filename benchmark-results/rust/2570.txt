{"title":"The Edge-First Manifesto","author":"Marco Bambini","date":"2025-12-19T10:03:07+00:00","main_content":"Cloud as a coordination layer. Edge as the execution environment.\n\nModern software is built on a fragile assumption: that the network is always available.\n\nIt isn’t.\n\nTrains lose connectivity.\nFactories operate behind firewalls.\nHospitals isolate systems.\nDevices move, disconnect, and fail.\n\nAnd yet we keep building applications that stop working the moment the cloud disappears.\n\nThis is not a limitation of hardware.\nIt is a limitation of mindset.\n\nThe cloud is not the computer\n\nFor the last decade, we have treated the cloud as the default execution environment.\n\nComputation, storage, identity, and application logic are centralized, always online, and always remote.\n\nThis model breaks under real-world conditions.\n\nLatency is unpredictable.\nCosts grow without bounds.\nPrivacy is compromised by design.\nFailure becomes systemic instead of local.\n\nThe cloud is a powerful coordination layer.\nIt is a terrible place to depend on for basic application behavior.\n\nSoftware must survive reality\n\nApplications should not stop working when connectivity degrades.\nThey should not lose data because a service is unavailable.\nThey should not require a round trip to a distant server to make progress.\n\nAn application that cannot function offline is not resilient.\nIt is incomplete.\n\nOffline is not an edge case.\nOffline is the baseline.\n\nEdge-first is not cloud-last\n\nEdge-first does not mean rejecting the cloud.\nIt means redefining its role.\n\nThe edge is where data is created.\nThe edge is where decisions must be made.\nThe edge is where users interact with software.\n\nThe cloud exists to coordinate, not to control.\n\nSynchronization should be explicit.\nConflicts should be deterministic.\nFailure should be local, not global.\n\nData belongs to the application, not the platform\n\nModern platforms encourage dependency.\n\nProprietary APIs.\nOpaque execution.\nIrreversible coupling.\n\nThis leads to lock-in, not reliability.\n\nApplications should be portable.\nState should be inspectable.\nLogic should be reproducible.\n\nA system you cannot debug locally is not under your control.\n\nSQLite is the natural foundation of the Edge\n\nSQLite is everywhere.\n\nIt is embedded, deterministic, battle-tested, and reliable.\n\nIt does not require a server.\nIt does not assume connectivity.\nIt does not hide complexity behind a network boundary.\n\nSQLite already powers the edge, silently.\n\nWe believe it should power the application runtime itself.\n\nAn application is a database with behavior\n\nStorage, computation, synchronization, security, and AI should not be separate services.\nThey should be capabilities of a single, local runtime.\n\nLogic runs where the data lives.\nState evolves deterministically.\nSynchronization happens when possible, not when required.\n\nThis is not a return to the past.\nIt is a correction.\n\nDeterminism over magic\n\nAutomatic systems that cannot be reasoned about eventually fail.\n\nWe choose explicitness over convenience.\n\nSynchronization must be observable.\nConflicts must be explainable.\nBehavior must be replayable.\n\nIf you cannot reproduce it, you cannot trust it.\n\nAI belongs on the device\n\nInference should not require a network round-trip.\nEmbeddings should not leak private data.\nLearning should respect locality.\n\nThe edge is not too weak for AI.\nThe cloud is too far.\n\nBuild software that works without permission\n\nSoftware should continue to function without connectivity, without subscriptions, and without centralized approval.\n\nThe best software does not ask for permission to exist.\n\nOur commitment\n\nWe are building an Edge Application Platform based on SQLite.\n\nOne that runs locally, synchronizes explicitly, executes logic on-device, integrates AI where data lives, and remains portable and open.\n\nThe cloud is optional.\nThe edge is mandatory.\n\nI am the founder of SQLite AI, we’re building an edge-first application platform based on SQLite, where apps run locally, work offline by default, and use the cloud only for coordination. Our goal is to treat SQLite as the application runtime, not just a storage layer.\nWe’ll be sharing more technical details and announcements soon."}
