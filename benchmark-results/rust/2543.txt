Overlay Loading overlay = Overlay(\"gpio_led.bit\")\n\nPYNQ's Overlay class:\nLoads the bitstream into the FPGA\nParses the.hwh file to understand the hardware structure\nCreates Python objects for each IP core (e.g., overlay.axi_gpio_0 )\n\nAXI GPIO Access led_gpio = overlay.axi_gpio_0.channel1\naxi_gpio_0 is the instance name from the Vivado block design\nchannel1 is the GPIO channel (Channel 1 is the first channel)\nThe GPIO object provides read() and write() methods\n\nLED Control led_gpio.write(pattern, mask)\npattern: Value to write (0x0 to 0x3 for 2 bits)\nmask: Bit mask indicating which bits to affect (0x3 = both bits)\n\nDesign Considerations\n\nPS-PL Interface Selection\nM_AXI_HPM0_LPD (Low Power Domain) is used instead of FPD (Full Power Domain)\nRationale: GPIO control requires low bandwidth, making LPD ideal for power efficiency\nLPD provides sufficient performance for GPIO operations while consuming less power\nFPD ports are better reserved for high-bandwidth applications (video processing, DMA, etc.)\n\nClock Domain\nPL clock: 100 MHz (from PS PL_CLK0)\nAXI bus clock: 100 MHz (synchronized with PL clock)\nGPIO update rate: Limited by Python loop speed (~4 Hz in this example)\n\nGPIO Configuration\nAll Outputs: GPIO configured as output-only (no input capability)\nWidth: 2 bits (matching two LEDs)\nI/O Standard: LVCMOS18 (compatible with KR260 HP bank)\n\nAddress Space\nAXI GPIO base address: 0x80000000 (LPD address space)\nAddress range: 64 KB (standard AXI4-Lite slave)\nAccessible from Linux userspace via PYNQ\nNote: LPD (Low Power Domain) uses the 0x80000000 address range, which is suitable for GPIO and other low-bandwidth peripherals\n\nNext Steps\n\nThis project provides a foundation for more advanced PL development:\nAdd More GPIO:
